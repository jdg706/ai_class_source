* 릴레이션 : 행과 열로 구성된 데이터(테이블)

      예)	스키마 : 데이터 구조에 대한 내용, 데이터 베이스
	각 데이터의 행 : 레코드 = 인스턴스 = 튜플
	각 데이터의 열 : 필드 = 컬럼 = 속성

######### 제약조건 #########

* 도메인 무결성 제약조건 : 지정된 자료형으로만 데이터가 입력, 수정 될 수 있도록 제한

* 개체 무결성 제약조건 : 기본키와 과련, 각 개체를 유일하게 구분할 수 있는 제약

* 참조 무결성 제약조건 : 부모 레코드에 자식 레코드가 있으면(참조하고 있으면) 부모 레코드를 삭제 할 수 없게 제한 (외래키 제약 조건)

######### 키 : 레코드를 대표할수 있는 값 #########

* 후보키 : 튜플을 [ 유일하게 식별 ] 할 수 있는 하나의 속성 혹은 속성의 집합, 속성의 최소 집합 (레코드를 대표할수 있는 값)

* 기본키(Primary Key) : 여러 [ 후보키 중에서 하나 ] 를 선정하여 대표하는 키

* 대리키 : 여러 후보키 중에서 기본키로 적합하지 않을 경우 일련번호 같은 형태로 [ 대신 ] 할 수 있는 키

* 외래키(Foreign key) : 다른 테이블의 데이터를 [ 참조 ] 하는 키
  - 관계형 데이터베이스의 가장 대표적인 특징
